Config-first, model-second. The connection block in config/database.php sets the large majority of behavior — adapter, auth driver, pagination strategy, filter dialect, endpoints, capabilities, guards, HTTP options. Models are the escape hatch: generated classes are committed code you own — narrow capabilities, override endpoint/resource type, remap names, add casts and relations. A model can tighten what config allows, never widen it.

How to read this diagram

Three bands. Build time (top): an artisan command reads a JSON spec file and writes real, physical model classes into your app — they're committed code, not runtime magic. Runtime (left): a normal Eloquent call flows down through the gated builder to HTTP and back. Config (right): one connection array wires nearly everything; the green dashed lines show what it controls.
